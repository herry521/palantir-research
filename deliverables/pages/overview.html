技术总览

这一页是研发团队的地图页。它的任务不是重复首页结论，而是把 Palantir 体系拆成可进入、可对照、可继续深挖的技术视图。

Platform Map

平台能力地图

如果把 Palantir 抽象成平台能力，而不是产品页面，表达层不能只看代码 DSL，还要把 transform / expression 算子基线纳入同一个能力图。

平台分层视角：更适合看“能力如何闭环”，而不是只看功能名词。
01

表达层

Transform DSL、Pipeline Builder、89 条 transform、335 条 expression、算子注册中心和类型化 IR，共同定义平台如何被开发者“描述”。

02

执行层

Spark、Faster、Flink、文件解析、地理空间和媒体处理等执行适配器，共同决定同一套语义计划如何稳定运行。

03

治理层

Dataset 事务版本、Branch、Lineage、Security、Marking 和质量门禁，让数据平台具备可追溯、可审计与可协作能力。

04

应用闭环层

Ontology、Writeback、AIP 和上层应用把“处理结果”变成“业务对象能力”，也是平台从数据系统走向业务系统的关键层。

Pipeline 开发入口与类型关系

这张关系图把开发入口和 Pipeline 类型拆开看：Pipeline Builder 与 Code Repositories 是 authoring surface；Batch、Incremental、Streaming 是 Pipeline 类型；Faster 是 Batch / Incremental 的变体。

Git、PR、测试、共享库和高码 Transform 工程入口。

Batch pipeline

Python SQL Java
Incremental pipeline

Python / Java 增量 API，用于更强 transaction 控制。

Python Java
Streaming pipeline

高级用户路径，主要面向 Java / UDF 和 streaming runtime 兼容。

Advanced Java + UDF
01

Streaming 不是 Batch 子类。Batch、Incremental、Streaming 应作为 Pipeline 类型并列。

02

Faster 不是独立类型。它是 Batch / Incremental 的执行变体，不覆盖 Streaming。

03

CodeRepo 支持 Stream。但属于 advanced users，不能等同于 Python Transform streaming。
