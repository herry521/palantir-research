核心理解 transform 更像 Pipeline Builder 中的“步骤”：它接收一个或多个数据集，完成聚合、连接、窗口、解析、展开、去重等数据处理逻辑，并输出新的数据集。expression 更像步骤里的“表达式”：它描述某个字段、某个值或某个结构该如何计算，常作为 transform 的参数或列级逻辑出现。

什么时候看 transform

当问题是“这批数据要怎样变形、合并、解析、聚合或排序”时，优先进入 transform 主线。典型例子包括 Aggregate、Pivot、Window、Mapping join、Parse Excel 和 Key by。

什么时候看 expression

当问题是“某一列该怎样计算、转换、清洗、判断或构造结构”时，优先进入 expression 主线。典型例子包括算术运算、条件分支、字符串处理、时间转换、JSON 解析和结构体更新。

第一层

数据集级转换

transform 关注输入数据集和输出数据集之间的变化，适合描述数据管道里的处理步骤。

第二层

字段和值级表达

expression 关注单个表达式如何计算结果，常被 transform 引用，用来描述列、条件、聚合表达式或结构化字段。

清单与场景并读

Transform 主线

理解数据集如何被处理成新的数据集

它解决什么问题

transform 描述 Pipeline Builder 中的数据处理步骤。它通常围绕数据集输入、参数配置和输出结果组织，用来表达聚合、连接、窗口、文件解析、地理处理、流式处理等场景。

怎样入门

先读 docs/transform-usage-examples.md，建立典型场景认知；需要查完整字段时，再读 artifacts/transform-final/README.md 指向的 JSONL、CSV 和 summary。

重点场景

Aggregate、Pivot、Window、Mapping join、Geometry intersection join、Parse Excel、Parse KML、First union by name、Key by 和 Rollup 是当前文档里的代表性入口。

清单特点

transform_inventory.csv 中的 supported_in、categories 和 declared_arguments 都按 JSON 结构序列化，便于人工查阅和程序处理。

Expression 主线

理解字段、值和结构如何被计算出来

它解决什么问题

expression 描述列级或值级计算逻辑。它可以完成数值运算、条件判断、空值兜底、字符串清洗、时间解析、类型转换、数组处理、结构体构造和 map 取值。

怎样入门

先读 docs/expression-usage-examples.md，理解常见表达式类别；需要查全量函数时，再读 artifacts/pb-expression-final/README.md 指向的 JSON、CSV 和 summary。

重点类别

Numeric、Boolean、Data preparation、String、Datetime、Cast、Array、Struct、JSON 和 Map 构成当前最值得先掌握的表达式地图。

清单特点

expression_inventory.csv 中的 category、supported_in、arguments 和 notes 都以 JSON 结构表达，适合快速筛选类别、参数和返回类型。
