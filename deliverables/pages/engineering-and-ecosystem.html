Topic 05

工程化与生态边界

真正让 Palantir 看起来像“完整平台”的，不只是功能本身，而是围绕测试、监控、安全、权限、AIP 与生态整合形成的工程体验。

Conclusions

结论

工程化成熟度决定平台“能不能长期用”

测试框架、构建历史、监控与告警、安全与权限，并不是附属能力，而是数据平台进入生产状态的最低保障。

AIP / MCP 体现的是控制面整合，而不是单点 AI 能力

真正有效的 AI 能力不是聊天式入口，而是能调用平台对象、上下文和执行能力的集成式助手。

Palantir 的开放性边界同样明显

私有血缘模型、封闭事务位点和较强的平台绑定，意味着它在跨平台互通上存在天然摩擦。

Media Set 是非结构化 AI 的资产入口

PDF、图片、音频、视频和 DICOM 不应只是附件，而应通过 Media Reference、Pipeline 转换和 Ontology 属性进入平台闭环。

Mechanisms

关键机制

1. 测试与研发流程被嵌入平台

TransformRunner、InMemoryDatastore、pytest 等测试路径，让 Transform 不必依赖真实 Dataset 才能验证逻辑，这是平台工程化的重要基础。

2. 可观测性与告警贯穿 Build 和 Stream

批处理关注构建成功率、耗时、资源消耗和失败恢复；流式则进一步关注端到端延迟、堆积、Checkpoint 和状态增长。

3. 安全与权限并不是单点 RBAC

从数据接入、Marking、字段权限、访问审计到审批流程，安全控制应随血缘与对象模型自然传播，而不是割裂在外围系统。

4. AIP / MCP 的价值在于“平台动作可被调用”

自然语言生成 SQL 或 PySpark 只是表层能力，真正的竞争力在于 AI 能读懂平台对象、上下文和执行路径。

5. 开放性短板值得正视

  • Lineage 未原生对齐 OpenLineage。
  • 增量位点没有对外暴露为通用消费接口。
  • 平台语义层优势和生态锁定风险同时存在。

6. 与开源栈的差异在于“整合深度”

开源生态可以覆盖很多单点能力，但需要大量控制面、元数据层和产品化整合工作，才能接近 Palantir 的整体体验。

Media Set Deep Dive

Media Set 如何把非结构化数据纳入平台

Media Set 不是 Dataset 的一个文件列，而是面向 PDF、图片、音频、视频、DICOM、邮件和表格文件的独立资产类型。它的核心链路是： Media Set -> Media Item -> Media Reference -> Dataset / Ontology / Application。

1. 文件资产化

Media Set 管理媒体 schema、path、RID、版本历史、retention 和上传策略；文件本体由 backing object store 或 virtual external source 承载。

2. 引用而非复制

Dataset 和 Ontology 保存 media_reference，下游应用通过引用解析文件，避免在 Pipeline、对象和应用中反复复制二进制内容。

3. Pipeline 批量转换

Pipeline Builder 和 Code Repository 可执行 OCR、PDF render、图片裁剪缩放、音频转写、DICOM 转表等转换，输出 Dataset rows 或新的 Media Set。

4. Ontology 展示与操作

业务对象通过 media reference property 挂载文档、影像、录音或视频；Workshop / Object Explorer / OSDK 读取同一个受控引用。

实现视角

Object Storemanaged / virtual
Media Setschema + item index
Media Referencestable RID pointer
Ontology / AIobject + VLM / OCR
  • 关键决策：path 只能做业务定位键，稳定引用必须使用不可变 media_item_id。
  • 事务边界：生产 Pipeline 输出优先使用 transactional write，避免失败后留下半成品。
  • AI 边界：OCR、转写、VLM 和 embedding 应作为可审计的 Media Transform，而不是让模型直接读对象存储路径。

Implications

对我们的启发

  • 测试、监控、安全、权限要与平台核心模型一体设计，而不是后置补丁。
  • AI 能力应优先服务平台动作和上下文调用，而不是先追求表面的自然语言演示效果。
  • Media Set 应作为和 Dataset 并列的资产模型预留，先打通 media_reference、转表、OCR 和 Ontology 属性，再扩展 VLM / embedding。
  • 在借鉴 Palantir 时，要清楚地区分“值得对标的整合能力”和“需要避免的过度封闭风险”。
  • 如果采用开源自建路线，最大的工作量会落在控制面、元数据层和工程化整合，而不是单个引擎接入。
