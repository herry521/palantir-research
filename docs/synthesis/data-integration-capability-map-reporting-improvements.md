# Data Integration 能力地图汇报页改进建议

## 摘要洞察

1. [判断] 当前页面更像“能力状态配置工具”，还不是“领导汇报页”：信息完整，但首屏缺少一句话结论、关键进展和风险指向。
2. [判断] 汇报页应从“列全能力”转为“讲清数据核心链路完备性”：用一条主链路承载叙事，把旁路能力降噪。
3. [建议] 最适合的形态是“PPT 封面页 + 一图总览 + 状态点亮 + 右侧洞察栏”的单页战情图，而不是纯模块卡片堆叠。
4. [建议] 二级/三级能力仍保留，但默认折叠或弱化，领导视角优先看到主链路成熟度、关键缺口、下一阶段动作。
5. [风险] 如果所有能力同等视觉权重，汇报时容易变成术语解释，而不是进展判断。

## 内容形式改进

### 1. 首屏增加“结论区”

建议在标题下方增加一句高密度结论，例如：

> Data Integration 已覆盖数据接入、加工、构建、调度、血缘与对象化主链路，当前核心能力完成 X%，主要缺口集中在实时链路与对象后端新鲜度保障。

这类结论要让领导不用读模块，也能知道当前状态。

### 2. 将能力地图改成“链路叙事”

当前结构是驱动层、主链路、支撑层，方向是对的。但汇报时建议把主链路视觉上表达为：

`Connect -> Land -> Transform -> Build -> Orchestrate -> Observe -> Serve`

对应 Palantir 能力口径可映射为：

- Connect: Data Connection / Agents / Sources / Sync / Streaming ingest
- Land: Datasets / Transactions / Streams / Branches
- Transform: Pipeline Builder / Code Repositories / Batch / Incremental / Streaming
- Build: Builds / Jobs / Build resolution / Staleness
- Orchestrate: Schedules / Triggers
- Observe: Data Lineage / Build timeline / Health / Quality
- Serve: Ontology / Object types / Object sets / Actions / Object Data Funnel

这样比按对象平铺更容易讲“数据从哪里来、如何加工、如何构建、如何服务业务”。

### 3. 给每个链路节点增加汇报级指标

当前统计是全局完成率、已完成、进行中、风险。建议每个主链路节点补充 1 个短指标：

- 进展：核心能力完成数
- 缺口：当前最大未完成项
- 影响：影响的数据链路或业务结果

示例：

> Build：4/4 核心能力完成；缺口：构建锁和强制构建策略待产品化；影响：异常重跑治理能力不足。

### 4. 区分“核心链路能力”和“旁路扩展能力”

领导汇报时不要让 Media sets、Virtual tables、Java transforms、External pipelines 等旁路能力抢主视觉。建议：

- 核心链路能力：默认显示，参与主完成率。
- 扩展能力：折叠、灰阶、或放入“补充能力池”。
- 风险能力：无论核心或扩展，都可以被突出。

### 5. 增加“下一阶段动作”

页面底部或右侧建议增加 3 条行动项：

- 近期补齐：1-2 个明确模块。
- 关键风险：1 个必须向上沟通的问题。
- 依赖资源：1 个跨团队或技术依赖。

这会让页面从“现状展示”变成“决策材料”。

## 页面样式改进

### 1. PPT 化首屏

当前首屏标题区域偏文档化。建议改成更像优秀 PPT 的封面页：

- 大标题保留，但副标题缩短到一行。
- 左侧放一句结论，右侧放 3 个关键数字。
- 背景保持克制，不要大面积装饰。
- 标签改成中文决策标签，例如“官方口径 / 可配置进度 / 领导汇报版”。

### 2. 强化“一图总览”

当前模块卡片太多，首屏只能看到驱动层开头。建议主图区域做成横向链路：

- 每个节点是紧凑卡片。
- 节点之间用箭头或连接线表达依赖。
- 完成状态用色条、角标、进度环表达。
- 点击或展开时再看二级/三级能力。

### 3. 右侧增加洞察栏

优秀汇报页通常不是只给图，还给“怎么看”。建议固定右侧或顶部增加洞察栏：

- 当前判断：整体处于什么阶段。
- 关键进展：已点亮的能力说明什么。
- 主要缺口：风险和未完成集中在哪里。
- 管理动作：需要领导拍板或协调什么。

### 4. 降低交互控件存在感

“导出 JSON / 导入 JSON / 重置”是配置工具功能，不适合领导汇报首屏。建议：

- 演示模式下隐藏配置按钮。
- 配置按钮移到右上角设置菜单。
- 默认展示汇报视图，而不是操作视图。

### 5. 使用状态语义而不是纯数量

当前 “0 / 0 / 0 / 风险 0” 是工具统计。建议升级为：

- 主链路成熟度：X%
- 已打通链路：X / 7
- 关键缺口：N 个
- 风险集中区：模块名

这更符合领导读数习惯。

## 推荐改版方向

建议采用“董事会战情图”风格：

- 顶部：一句话结论 + 关键数字。
- 中部：横向数据链路能力地图，按状态点亮。
- 右侧：洞察与风险。
- 底部：下一阶段动作和依赖。

这个方向保留当前页面的配置能力，但默认呈现应服务汇报，而不是服务编辑。
