## 2. Q2 工作拆解（2026-04 ~ 2026-06）

### 2.1 月度交付节奏

| 月份 | 关键动作 | 主要输出 |
|------|---------|---------|
| **4月上旬** | 冻结演示范围，准备方案评审材料 | 演示场景清单、技术方案草稿 |
| **4月中旬** | 完成全链路方案评审 | Pipeline + Schedule + Build + OSS 四域方案评审通过，设计文档冻结 |
| **4月中旬起** | 启动 630 演示内容开发，同步开发 Pipeline Builder v1 | 演示 Demo 骨架、Pipeline Builder UI 框架 |
| **5月** | 演示内容主干开发，Pipeline Builder v1 核心功能开发 | 可运行演示 Demo，Builder 核心交互可用 |
| **5月底** | 演示内容开发收尾，进入联调准备 | 演示案例完整跑通（参考：[E2E Workflow Speedrun](https://learn.palantir.com/speedrun-your-first-e2e-workflow)） |
| **6月** | 系统联调测试，H1 演示功能交付 | H1 演示功能完整交付，联调测试报告，上线交接文档 |

### 2.2 核心工作包

| 工作包 | 负责域 | Q2 完成标准 |
|-------|-------|-----------|
| **全链路方案评审** | Pipeline + Schedule + Build + OSS | 四域技术方案评审通过，接口与数据契约冻结 |
| **630 演示内容开发** | 产品 + 工程 | 端到端演示案例完整可跑通，含数据接入→加工→消费全链路 |
| **Pipeline Builder v1 开发启动** | 前端 + 后端 | Builder 核心交互（DAG 编排、节点配置）在 Q2 内可 Demo |
| **系统联调测试** | 全链路 | 联调覆盖演示路径上所有节点，问题清零，H1 演示功能稳定 |

## 3. Q3 工作拆解（2026-07 ~ 2026-09）

### 3.1 月度交付节奏

| 月份 | 关键动作 | 主要输出 |
|------|---------|---------|
| **7月** | 完成 Pipeline Builder v1 交付，同步输出 930 产品原型 | Pipeline Builder v1 GA，930 产品原型文档与演示稿 |
| **8月** | 数据工程拓展：支持「文档」和「时序」数据类型 | 文档型数据接入 + 时序型数据接入，对应 Transform 算子可用 |
| **9月** | 面向 930 演示内容查漏补缺与体验优化 | 演示路径稳定，体验问题清零，I1 全链路验收通过 |

### 3.2 核心工作包

| 工作包 | 负责域 | Q3 完成标准 |
|-------|-------|-----------|
| **Pipeline Builder v1 正式交付** | 产品 + 工程 | Builder 功能完整，文档齐全，可作为主力开发工具上线 |
| **930 产品原型输出** | 产品 | 完整产品原型，含演示剧本、功能清单、交互稿 |

| **930 演示体验优化** | 全链路 | 对标演示案例进行系统性查漏补缺，体验问题清零 |

## 5. 风险与前置依赖

1. **方案评审阻塞风险**：全链路方案评审若 4 月中旬无法通过，将直接影响 5-6 月的开发节奏与演示交付。
2. **演示范围与 Builder 并行风险**：630 演示开发与 Pipeline Builder v1 同步进行，需明确资源分配与优先级切换规则。
3. **数据类型拓展依赖风险**：文档型和时序型数据支持涉及底层存储扩展，需在 Q3 初（7月）完成技术方案冻结。
4. **9月查漏时间压缩风险**：若 8 月数据类型拓展出现延期，将直接压缩 9 月演示优化窗口。
